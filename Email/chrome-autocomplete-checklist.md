# Чек-лист: автокомплит email/пароля и промпт "Сохранить пароль?" в Chrome

## 1. Код формы (Vue)

- [ ] Поля email и password находятся внутри настоящего `<form>`, а не просто в `<div>`.
- [ ] У email-поля атрибут `autocomplete="username"` (не `"email"` — это другая категория, "Адреса", не Password Manager).
- [ ] У поля пароля атрибут `autocomplete`:
  - `current-password` — для логина существующего аккаунта;
  - `new-password` — для регистрации/смены пароля.
- [ ] Поле пароля **не удаляется из DOM** между шагами (`v-if`), а скрывается визуально (`v-show`/CSS). Если пароль на момент рендера формы отсутствует в DOM, Chrome не распознаёт форму как логин-форму и не привязывает к ней Password Manager.
- [ ] Форма обрабатывает submit нативно: `<form @submit.prevent="onSubmit">` + кнопка `type="submit"`.
- [ ] **Только финальный шаг (логин/регистрация с заполненным паролем) должен триггерить настоящий `submit` формы.** Промежуточные шаги (например, "ввести email и проверить, есть ли аккаунт") должны использовать кнопку `type="button"` с обычным `@click`, а не `type="submit"`. Если форма отправляется (submit-событием) несколько раз за один визит — в первый раз с пустым паролем, во второй с настоящим, — Chrome путается и может вообще не предложить сохранить пароль.
- [ ] После успешного логина должно произойти что-то похожее на "навигацию" (`window.location.reload()`, переход на другую страницу и т.п.) — это сигнал для Chrome, что попытка входа была успешной.

## 2. Окружение браузера при тестировании

- [ ] DevTools **Device Toolbar** (responsive/emulation режим) выключен. Если он активен, Chrome может не показывать нативные UI-промпты (включая "Сохранить пароль?").
- [ ] Сайт открыт по `https://`, а не по `http://`.
- [ ] Тестирование происходит не в Incognito/гостевом режиме (там сохранение паролей отключено по умолчанию).
- [ ] Нет конфликтующих расширений-менеджеров паролей (1Password, Bitwarden, LastPass и т.п.), которые могут перехватывать форму и подавлять нативный промпт Chrome.

## 3. Настройки Chrome (актуальные адреса)

Старый `chrome://settings/passwords` в новых версиях Chrome заменён на отдельное приложение Password Manager:

- [ ] **`chrome://password-manager/settings`** — тоггл "Offer to save passwords / Предлагать сохранение паролей" включён.
- [ ] **`chrome://password-manager/passwords`** — проверить:
  - нет домена в списке "Never saved" (если раньше нажал "Никогда" для этого сайта во время тестов — Chrome больше не предложит, пока запись не удалить);
  - нет уже сохранённой записи с тем же email/паролем для этого домена (иначе Chrome просто тихо залогинит без промпта, т.к. совпадение уже сохранено).
- [ ] **`chrome://management`** — браузер не managed организацией (если managed — Password Manager может быть отключён политикой IT, и это не лечится на уровне пользователя/кода).

## Итог

Промпт "Сохранить пароль?" и автокомплит сохранённых логинов зависят и от кода формы, и от состояния браузера/профиля. Если код правильный, но не работает — сначала проверять пункты раздела 2 и 3, а не сразу менять код.
